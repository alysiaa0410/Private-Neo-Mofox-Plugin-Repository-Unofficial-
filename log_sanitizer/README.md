## 日志隐私脱敏插件（log_sanitizer）

### 功能

在调试与记录日志时，对可能包含手机号、邮箱、身份证号等敏感信息的文本进行简单脱敏处理，降低日志意外泄露用户隐私的风险。

### 指令

- `/sanitize <文本内容>` - 对文本进行脱敏处理

### 权限节点

- `log_sanitizer.use` - 允许使用脱敏功能

### 使用示例

```
/sanitize 我的号码是 13812345678，邮箱 test@example.com
🔐 脱敏结果：
原文：我的号码是 13812345678，邮箱 test@example.com
脱敏：我的号码是 138****5678，邮箱 t**t@example.com
```

### 支持的脱敏类型

- 手机号：保留前3位和后4位，中间用 **** 替代
- 邮箱：保留首尾字符和域名，中间用 * 替代
- 身份证号：保留前6位和后4位，中间用 ******** 替代
