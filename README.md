## 自制 MoFox-Bot 插件合集（7 个插件）

这是一个面向新手的 **MoFox-Bot 插件合集/示例仓库**：每个插件一个文件夹，结构参考官方模板，方便你直接复制成独立仓库并提交到官方插件索引。

所有插件均基于官方模板风格：每个插件独立一个文件夹，包含 `__init__.py`（元信息）和 `plugin.py`（核心逻辑），并各自附带独立的 `README.md`。

### 你可以怎么用这个仓库

- **作为“插件点子库”**：直接挑一个功能，把对应文件夹复制到你自己的独立插件仓库里。
- **作为“插件脚手架”**：用 `tools/scaffold_mofox_plugin.py` 一键生成新插件目录。
- **作为“规范参考”**：本仓库包含 `.github/workflows/validate.yml`、`tools/validate_repo.py`，用于自动化检查目录结构与基础规范。
- **作为“发布辅助工具箱”**：
  - `tools/export_plugin_repo.py`：把某个插件导出为“独立插件仓库”目录结构；
  - `tools/export_all_plugins.py`：一键把 7 个插件全部导出到 `exported/`，做到“复制上传就行”；
  - `tools/generate_registry_entries.py`：生成可提交到官方索引的 `plugins.json` / `plugin_details.json` 草稿（输出到 `registry/`）。

### 快速自检（可选但推荐）

在仓库根目录执行：

```bash
python tools/validate_repo.py
```

如果你想做格式化和静态检查：

```bash
python -m pip install -r requirements-dev.txt
ruff check .
ruff format .
```

### 1. 对话情绪分析与安全建议插件（`conversation_safety_guard`）

- **定位**：基础版“情绪雷达”，通过简单关键词对一段文本做情绪倾向判断，并给出温和的安全建议。
- **核心指令**：  
  - `/mood 文本内容` / `心情检测 文本内容`
- **典型场景**：  
  - 想快速判断用户当前是偏积极、中性还是消极/高风险情绪；  
  - 为后续接入更专业的情绪识别/陪伴功能提供一个简单起点。

---

### 2. 轻量 FAQ 与知识库增强插件（`simple_faq_knowledge`）

- **定位**：不依赖外部服务的超轻量 FAQ / 知识库，专门回答“常见问题”和“使用说明”。
- **核心指令**：  
  - `/faq 关键字...`
- **典型场景**：  
  - 为 Bot 补充“自助帮助文档”“常见问题”；  
  - 希望对某些问题给出稳定、一致的回答，而不是每次都走大模型。

---

### 3. 多角色预设与场景切换插件（`scene_persona_switcher`）

- **定位**：多种人设/场景一键切换的“人设管理器”。
- **核心指令**：  
  - `/scene list` 查看所有预设场景；  
  - `/scene 学习`、`/scene 吐槽`、`/scene 严肃` 等切换不同模式。
- **典型场景**：  
  - 一个 Bot 既要能“认真讲题”，又要能“日常闲聊”，需要根据场景切换提示词和语气；  
  - 与 `custom_prompt_injector` 等 Prompt 注入插件配合使用。

---

### 4. 亲密度与成就系统插件（`affinity_achievements`）

- **定位**：给 Bot 加上一层“好感度 + 成就称号”的轻度游戏化系统。
- **核心指令**：  
  - `/affinity` 查看当前亲密度与已解锁成就。
- **典型场景**：  
  - 想让长期陪伴型 Bot 更具“成长感”，例如解锁“老朋友”“形影不离的搭子”等称号；  
  - 给常来聊天的用户一点正向反馈和收集乐趣。

---

### 5. 日志隐私脱敏插件（`log_sanitizer`）

- **定位**：演示与实践“日志脱敏”思路的工具插件。
- **核心指令**：  
  - `/sanitize 文本内容`
- **功能要点**：  
  - 对文本中的手机号、邮箱、可能的身份证号片段做基础掩码；  
  - 可直接复用 `sanitize_text` 函数集成到你自己的日志中间件中。

---

### 6. 对话简要回顾插件（`conversation_recap`）

- **定位**：基于近期消息生成一个“对话小结”，方便用户快速回忆聊了什么。
- **核心指令**：  
  - `/recap`
- **典型场景**：  
  - 用户短暂离开后回来，想快速知道“刚才聊到哪了”；  
  - 与更高级的“日记式长期记忆”插件搭配使用，作为轻量版本。

---

### 7. 每日签到与连击插件（`daily_checkin`）

- **定位**：简单的“每日签到 + 连击天数”系统，增强日常互动黏性。
- **核心指令**：  
  - `/checkin` 今日签到；  
  - `/checkin status` 查看当前连击天数。
- **典型场景**：  
  - 想给 Bot 增加一点“运营活动”和打卡感；  
  - 和亲密度系统结合，让“坚持来聊天”的用户获得更高好感加成。

### 如何在 MoFox 插件生态中使用这些插件

1. **拆分成独立仓库**  
   为上述每一个插件文件夹单独创建一个公开 GitHub 仓库，并复制对应目录下的内容（含 `README.md`）。

2. **在官方插件索引中登记**  
   在 [`MoFox-Plugin-Repo`](https://github.com/MoFox-Studio/MoFox-Plugin-Repo) 的 `plugins.json` 与 `plugin_details.json` 中添加对应的 `id` 和 `manifest` 信息（示例已在对话中给出）。

3. **在自己的 MoFox-Bot 部署中启用**  
   将这些插件仓库地址加入到你的插件配置中（按 MoFox 官方文档的插件加载方式），重启 Bot 即可使用这些指令与能力。
